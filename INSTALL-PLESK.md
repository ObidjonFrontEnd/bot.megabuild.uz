# Установка бота на Plesk (без SSH, через панель)

Инструкция для хостинга Plesk, где нет терминала. Всё делается мышкой в панели.
Скрипт `install.sh` здесь **не нужен** — он только для серверов с SSH.

Что используем из панели: **Git**, **PHP Composer**, **выбор версии PHP**,
**смена корня сайта (Document Root)**. Node.js для бота не нужен.

---

## Шаг 1. Создать домен/поддомен для бота

В Plesk: **«Сайты и домены» → «Добавить поддомен»** (например `bot.ваш-домен.uz`).

Сразу включите для него **SSL (Let's Encrypt)**:
**«Сайты и домены» → SSL/TLS-сертификаты → Установить бесплатный сертификат**.
Telegram работает только по `https`.

## Шаг 2. Выбрать PHP 8.2+ и включить расширения

**«Сайты и домены» → выбранный поддомен → «Настройки PHP»:**
- Версия PHP: **8.2 или новее**.
- В списке расширений убедитесь, что включены: **pdo_sqlite (или sqlite3), curl, mbstring**.
  Обычно они уже включены.

## Шаг 3. Скачать проект через Git

**«Сайты и домены» → поддомен → Git:**
1. Нажмите **«Добавить репозиторий Git»**.
2. Вставьте адрес вашего репозитория.
3. В качестве папки развёртывания оставьте предложенную (например `httpdocs`)
   или укажите свою — запомните этот путь, он понадобится дальше.
4. Нажмите **«Развернуть»** (Deploy) — Plesk скачает файлы.

> В репозитории сам бот лежит в подпапке **`bot/`**. То есть после клонирования
> путь к боту будет, например: `httpdocs/bot`.

## Шаг 4. Установить зависимости (Composer)

**«Сайты и домены» → поддомен → PHP Composer:**
1. Plesk попросит указать папку, где лежит `composer.json` — это папка **`bot`**
   (например `httpdocs/bot`).
2. Нажмите **«Установить»** (Install). Plesk выполнит `composer install` сам.

Если PHP Composer просит выбрать действие — выбирайте **install**.

## Шаг 5. Создать файл .env и вписать 3 значения

**«Сайты и домены» → поддомен → Файловый менеджер**, зайдите в папку `bot`:
1. Найдите файл **`.env.example`**, скопируйте его и переименуйте копию в **`.env`**.
2. Откройте `.env` на редактирование и заполните три строки:

```
TELEGRAM_TOKEN=токен_от_BotFather
YII2_API_BASE_URL=http://46.62.246.156/api      ← адрес API основного сервера, без / в конце
APP_URL=https://bot.ваш-домен.uz                ← адрес этого бота
```

Строку `DB_PATH=` оставьте **пустой** — база создастся сама в папке `data/`.

## Шаг 6. Указать корень сайта на папку `bot/web`

Это самый важный шаг. **«Сайты и домены» → поддомен → «Настройки хостинга»**
→ поле **«Корневой каталог документов» (Document Root)**:

```
httpdocs/bot/web
```
(подставьте свой путь из шага 3, но обязательно с `/bot/web` в конце).

Сохраните. В папке `web` уже лежит `.htaccess` с нужными правилами.

## Шаг 7. Зарегистрировать webhook Telegram

Без SSH это делается **через браузер** — просто откройте один раз такую ссылку,
подставив свой токен и домен:

```
https://api.telegram.org/bot<ВАШ_ТОКЕН>/setWebhook?url=https://bot.ваш-домен.uz/webhook
```

Пример (с выдуманным токеном):
```
https://api.telegram.org/bot123456:AAH.../setWebhook?url=https://bot.ваш-домен.uz/webhook
```

В ответ должно прийти: `{"ok":true,"result":true,"description":"Webhook was set"}`.

## Готово 🎉

Откройте бота в Telegram и отправьте `/start`.

## Если Mini App не загружает данные (про /api/)

Mini App обращается к адресу бота `/api/...`, а его нужно переслать на основной
Yii2-сервер. На Plesk это настраивается так:

**«Сайты и домены» → поддомен → «Apache и nginx» → «Дополнительные директивы nginx»**
— вставьте (замените адрес на ваш Yii2-сервер):

```nginx
location /api/ {
    proxy_pass         http://46.62.246.156/api/;
    proxy_http_version 1.1;
    proxy_set_header   Host              $host;
    proxy_set_header   X-Real-IP         $remote_addr;
    proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header   X-Forwarded-Proto $scheme;
    proxy_read_timeout 30s;
}
```
Сохраните — Plesk сам перезагрузит nginx.

## Обновление до новой версии

Когда в репозиторий зальёте изменения:
1. **Git → «Развернуть» (Deploy)** — Plesk подтянет новый код.
2. **PHP Composer → «Обновить» (Update/Install)** — если менялись зависимости.

Файл `.env` и база `data/bot.sqlite` при этом не трогаются.

---

## Если что-то не работает

| Симптом | Решение |
|---|---|
| `Database Exception (#14)` | Папка `bot/data` недоступна для записи. В Файловом менеджере проверьте права на папки `data` и `runtime` (должны быть доступны на запись владельцу сайта). |
| Открывается список файлов / 403 / 404 | Корень сайта не указывает на `bot/web` (шаг 6) либо не сработал `.htaccess`. |
| Бот молчит на `/start` | Не зарегистрирован webhook (шаг 7) или домен без https. |
| `Class not found` / белый экран | Не выполнен Composer (шаг 4). |
| Mini App не грузит данные | Неверный `YII2_API_BASE_URL` в `.env` или не добавлен блок `/api/` (см. выше). |
